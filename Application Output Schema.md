**Application Output Schema**

**Purpose**

This schema defines the exact output contract for the application
customization workflow.

It is designed to keep the workflow low-friction and immediately usable.

It defines only:

- required output sections

- section order

- formatting rules

- length / content constraints

- what is primary vs secondary

It does not define workflow logic, resume-editing logic, or cover-letter
voice rules.

**Primary design rule**

The primary resume deliverable must be a **Full Updated Resume Draft**.

The workflow should not default to step-by-step edit instructions.

**Section order**

1.  Resume Base Used

2.  Targeting Notes

3.  Full Updated Resume Draft

4.  Optional Change Notes

5.  Cover Letter

6.  Residual Risks / Gaps

**Section definitions**

**1. Resume Base Used**

**Purpose\**
State which resume served as the base.

**Format\**
One line only.

**Allowed values**

- Master Resume

- Variant A

- Variant B

- Variant C

**2. Targeting Notes**

**Purpose\**
Briefly state what the resume was optimized for.

**Format\**
2--4 short bullets maximum

**Should include**

- target function

- main emphasis shifts

- major constraints if relevant

**Should not include**

- long rationale paragraphs

- detailed process explanation

**3. Full Updated Resume Draft**

**Purpose\**
Provide the fully tailored resume itself.

**Format**

- full resume text

- ready to review and refine

- not just instructions

- preserve resume-style formatting as cleanly as possible in plain text

**Rule\**
This is mandatory.

**4. Optional Change Notes**

**Purpose\**
Provide concise notes on the most important changes, if useful.

**Format\**
Short bullets only

**Length\**
Maximum 6 bullets

**Rule\**
Optional. Include only when they materially help review.

**5. Cover Letter**

**Purpose\**
Provide the tailored cover letter draft.

**Format**

- full letter text

- aligned with the Cover Letter Voice Guide

- ready to review and refine

**Rule\**
Include when requested by the user or by the active application task.

**6. Residual Risks / Gaps**

**Purpose\**
Surface the most important remaining weaknesses or uncertainties after
customization.

**Format\**
Short bullets only

**Length\**
Maximum 5 bullets

**Rule\**
Only meaningful items. No filler.

**General output rules**

- Do not replace the full resume draft with edit instructions.

- Keep change notes secondary.

- Keep review friction low.

- Do not add extra sections unless requested.

- Keep residual risk comments concise and useful.

- Do not repeat the same point in Optional Change Notes and Residual
  Risks / Gaps.

- The output should feel directly usable, not like homework for the
  user.
